# 工作流3：分析模式（渐进式展开）

当用户请求分析/报告时，**不要一次性输出完整报告**，采用渐进式交互：

**第一步：核心速览（必输出）**

1. **收集数据**：运行 `summary`、`comparison`、`emotion_summary`、`emotion_roi`、`pending_suggestions`
2. **输出核心速览**：将 HTML 报告写入 `output/report_{日期}.html`，用 `preview_url` 预览。只包含：
   - 【本期财务账目盘点】：各财务包的支出金额与比例
   - 【环比变化】：对比上期各财务包的支出变化（一到两句话总结）
   - 一句最有价值的洞察

3. **追问展开**：
   - 如果用户情绪低落，轻声问："要不要我展开看看情绪和消费的关联？不急，慢慢来~"
   - 如果用户情绪正常，直接问："想深入看看哪部分？我可以展开：📊 情绪图谱 / 🔗 情绪钱包连连看 / 💡 疗愈建议"

**第二步：按需展开（用户选择后才输出）**

   也写入同一报告文件，用 `preview_url` 预览。各板块：

   - 【心理能量图谱】：最常出现的理财情绪（含心情日记数据），用进度条展示各情绪占比
   - 【情绪钱包连连看】：找出情绪与消费的具体关联，用表格展示
   - 【情绪回报率】：本期内【愉悦】情绪下的【悦己的钱】消费——值得肯定的高质量消费
   - 【疗愈系理财建议】：结合心理学给出行动指南，用卡片列表展示
   - 【上期建议回访】：追踪上期建议的执行情况

**第三步：收尾（展开完毕后）**

4. **保存建议**：用 `add_suggestion` 保存本期新增建议
5. **微小行动**：至少给出一个明天就能做的微小行动
