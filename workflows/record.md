# 工作流1a：记账模式（单一消费）

当用户输入包含金额、消费项目时：

1. **识别财务数据**：提取项目名称、金额、类型（支出/收入）
2. **归类财务包**：参考 `references/finance_pockets.md` 中的归类规则
3. **识别情绪**：参考 `references/emotion_system.md` 匹配情绪标签
4. **情绪追问策略**：
   - 【要命的钱】日常刚需 → **不追问**，自动标记"日常"
   - 【悦己的钱】→ 如果用户没写心情，**温柔追问**
   - 【生钱的钱】/【保命的钱】→ 视情况追问
   - 用户已主动写了心情 → 直接记录
5. **运行命令**：`echo '<JSON>' | python3 $SCRIPT add_bill`
6. **检查预算**：运行 `budget_usage` 检查使用率，必要时预警
7. **确认输出**：将 HTML 写入 `output/` 目录，用 `preview_url` 预览。单条或多条记录合并为一张卡片

   **财务包标签配色**：🏠 要命的钱=蓝底(#dbeafe/#1e40af)、🛡️ 保命的钱=绿底(#dcfce7/#166534)、📈 生钱的钱=橙底(#ffedd5/#9a3412)、🎁 悦己的钱=粉底(#fce7f3/#9d174d)

   [如果有预算预警，追加提醒]
   [如果有待确认心情，追问]

# 工作流1b：拆分记账模式（复合消费）

当用户描述一笔消费包含多个类别时（如"去超市买了卫生纸和零食，一共86块"）：

1. **识别复合消费**：判断是否涉及多个财务包
2. **询问拆分**：主动建议拆分方式，例如：
   "老大，这笔消费看起来包含不同类别，我帮你拆开记吧：卫生纸约￥26→【要命的钱】，零食约￥60→【悦己的钱】，这样拆可以吗？"
3. **用户确认后**，使用 `splits` 字段一次性记录：
   ```json
   {
     "item": "超市购物",
     "amount": 86,
     "type": "expense",
     "pocket": "悦己的钱",
     "splits": [
       {"item": "卫生纸", "amount": 26, "pocket": "要命的钱", "emotion_tag": "日常"},
       {"item": "零食", "amount": 60, "pocket": "悦己的钱", "emotion_tag": "愉悦"}
     ]
   }
   ```
4. 如果用户不想拆分，按金额最大的项目归类整笔消费
